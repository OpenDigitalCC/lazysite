# Non-functional report — lazysite (SM070 WebDAV)

Date: 2026-06-12
Scope: changed areas for SM070 (WebDAV endpoint + per-user access
mechanisms). Mid-feature close-out, not a full-project audit, per
`rules/nonfunctional-close.md`.
Auditor: Claude

## 1. Test coverage

**Method.** `prove -r t/` (883 green). Coverage measured with
`Devel::Cover 1.44`, propagating instrumentation into the
subprocess-run scripts via `PERL5OPT=-MDevel::Cover` with an absolute
shared db. The SM070 test set (`t/unit/dav`, the new users/auth/manager
tests, the integration + journey tests) was run under coverage.

**Findings.**

- Suite grew from 684 to **883 tests** (all green), +199 across 13 new
  files: `t/unit/users/{05-settings,06-token}.t`,
  `t/unit/auth/04-ui-flag.t`, `t/unit/dav/01..07`,
  `t/unit/manager/08-lock-interop.t`, `t/integration/dav-publish.t`,
  `t/journey/05-webdav-publish.t`.
- **`lazysite-dav.pl` (the new file): 88.8% statement, 98.5%
  subroutine, 67.3% branch.** Statements and subroutines clear the
  ≥80% target. The single uncovered subroutine is `_json_str` (only
  reached when `LAZYSITE_LOG_FORMAT=json`). Uncovered branches are
  error/defensive paths: 500s on `open`/`rename`/`close` failure, the
  fail-open `DB_File` tie branches in the rate limiter, and the
  cross-device MOVE fallback (`copy_tree` + unlink) — none cheaply
  triggerable in a unit test.
- **`tools/lazysite-users.pl`: 92.3% statement, 94.2% subroutine**
  across its full test set (`01`/`02`/`05`/`06`) — the new
  settings/token/guard code is well covered.
- `lazysite-manager-api.pl` whole-file coverage is 42.6% statement —
  the pre-existing level for this large, largely manually-tested file
  (see "Manager API end-to-end via browser" below); the SM070 lock-sub
  changes specifically are covered by `08-lock-interop`.
- Every endpoint method is exercised, plus the gate chain, path
  containment (traversal / internal-tree / blocked / scope / symlink),
  conditionals, locking (incl. refresh, flood guard, owner escaping),
  and the manager↔DAV lock interop in both directions. Negative paths
  are the bulk of the DAV suite — the executable form of the §4
  controls.

**Gaps / recommendations.**

- `lazysite-auth.pl` coverage was not captured: its subprocess tests
  (`04-ui-flag`, journey) build a clean `%ENV`, which drops the
  `PERL5OPT` that carries the coverage instrumentation. The new `ui`
  path is verified by 14 passing assertions; to get a coverage number,
  have those tests preserve `PERL5OPT` when they replace `%ENV`.
- Branch coverage on `lazysite-dav.pl` (67.3%) is the honest weak
  spot. A handful of fault-injection tests (unwritable target dir,
  forced `rename` failure) would lift it; the uncovered branches are
  all error returns, not logic.
- `litmus` is not wired up (binary absent). Recommend a
  `t/integration/litmus.t` that runs when `LITMUS_BIN` is set, pinning
  `basic`/`copymove`/`locks` pass and the expected `props` failures.
  Until then, desktop-mount (Explorer/Finder) interop is unverified by
  automation — verify manually before relying on it.

## 2. Code quality

**Method.** `perl -c` on all changed scripts; `perlcritic --severity 3`
compared against the existing `lazysite-manager-api.pl` baseline.

**Findings.**

- All changed Perl compiles clean; shell (`commit.sh`) passes `bash -n`.
- `lazysite-dav.pl`'s perlcritic profile matches the existing
  manager API: the violations are the same policy classes the codebase
  already accepts — `RequireExtendedFormatting` (no `/x`, documented
  deviation), `RequireFinalReturn` on dispatch subs, `ProhibitExplicit
  ReturnUndef`, per-sub complexity on the method handlers, and the
  `:utf8` layer (documented deviation). No new policy class is
  introduced.
- Two flagged items were checked and are false positives: `$1` after a
  `while (m//g)` (always defined in the loop body) and a `> 0` test
  inside an `unless`.
- Self-contained-script policy honoured; the duplication-by-convention
  list in `docs/architecture/code-quality.md` is updated, and the
  shared lock-record format is documented as a deliberate cross-script
  contract.

**Recommendations.**

- The `do_*` method handlers and `main` carry real cyclomatic
  complexity (24–30). They are flat and grep-able, matching the
  manager API's idiom, so refactoring is optional — but if the lock
  rules grow, factor the write-precheck (conditionals + lock guard)
  into one helper used by all write methods (already partially done).

## 3. Performance

**Method.** Reasoned against the documented CGI baseline (no profiler
run for this feature).

**Findings.**

- Auth overhead with a generated credential is one SHA-256
  (`iterations=1`); a human password is 100k iterations per request —
  documented as the reason to use generated credentials for
  automation.
- PUT streams the body in 64 KiB chunks; memory is bounded regardless
  of file size. The `CONTENT_LENGTH` gate rejects oversize uploads
  before reading.
- PROPFIND is Depth 0/1 only (no recursive walks); cost is O(entries)
  with one `stat` + one lock lookup each.

**Recommendations.**

- Capture a measured depth-1 PROPFIND time against a ~1000-entry
  directory and add a DAV row to `performance.md`'s baseline table
  (placeholder text added; numbers pending a measurement host).

## 4. Security

**Method.** `perlcritic --theme=security`; manual trust-boundary
review against the §4 threat model; diff secrets scan.

**Findings.**

- Security-theme critic returns only the `:utf8`-layer items (a
  documented, deliberate deviation) — no injection, eval, or unsafe
  `open`/`system` findings.
- Every filesystem call in `lazysite-dav.pl` is preceded by the
  resolve→internal-tree→blocked→scope chain (including the
  LOCK-creates-file path, which runs the full write chain). `realpath`
  + docroot-prefix catches symlink escape on source and destination.
  PATH_INFO is not re-decoded (no double-decode traversal); the
  `Destination` header is decoded and re-validated.
- Auth: Basic over TLS only (plaintext refused except loopback /
  explicit `dav_allow_insecure`); per-IP failed-attempt limiter;
  proxy `X-Remote-*` / `LAZYSITE_AUTH_TRUSTED` ignored (test-pinned).
- Locks: tokens from the fail-closed CSPRNG; UNLOCK requires token
  **and** owner; manager-origin locks not overridable from DAV; owner
  XML truncated + escaped at every echo (no stored-XSS into the
  manager UI); per-user lock-count cap.
- `ui:off` is enforced post-verification (no oracle) and blocks cookie
  issuance, gating manager UI / API / protected pages together;
  localhost no-password bypass respects it.
- `user-settings.json` is in the manager `@BLOCKED_PATHS`; the whole
  `lazysite/` tree is unreachable over DAV.
- Diff secrets scan clean: no `lzs_` literals or fixture plaintext
  credentials committed (tests generate at runtime).

**Recommendations.**

- No per-user WebDAV upload quota this release (accepted residual
  risk, consistent with the manager API). If abuse appears, reuse the
  manager upload-quota machinery.
- `gitleaks`/`trufflehog` not installed here; run one over the diff in
  CI as belt-and-braces.
