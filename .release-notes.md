Release 0.4.47 - Audit search/targets + plugin discovery + page split

A run of operator-reported fixes plus a manager restructure. Full suite green; perlcritic
clean; strict SBOM gate green.

## Audit: better targets and a date-range search

- nav-save and plugin-enable/disable audited a bare '/' as their target; they now name
  what they touched (nav; the plugin id/name). The auth lifecycle (claim/forgot/token/
  mfa/oauth) was already covered.
- The Audit page gains From / To date filters - action_audit filters by an ISO timestamp
  range (a bare date widens to the whole day). Either bound is optional.

## Plugins are discovered dynamically

action_plugin_list used a hard-coded candidate list, so a newly added plugin (the SM083
Visitor Stats plugin) never appeared - "Visitor Stats plugin is not installed". It now
scans plugins/*.pl, each still validated by --describe, so any future plugin shows up
with no code change.

## Plugin Manager vs Plugin Config

Plugin management is split across two pages, per operator request:

- Plugin Manager (/manager/plugins) - enable/disable, moved off the Configuration page.
- Plugin Config (/manager/plugin-config) - the per-plugin settings UI (formerly the
  /manager/plugins page).

The Configuration page now carries only site settings and links out; the nav and command
palette list both pages; an enabled plugin has a Configure link straight to its settings.
