## UTF-8 corruption in JSON responses (important)

send_json (MCP), respond (control API), respond_json (OAuth) and the manager-api
error/preview responders printed encode_json's output - which is ALREADY UTF-8
bytes - under a :utf8 STDOUT layer, re-encoding it. So non-ASCII (+/-, pound,
accents, en-dashes, curly quotes) came back as mojibake on read / preview. The
write path stored correctly; the read RESPONSE was corrupting. Now they print the
bytes raw; HTML/XML responders (processor, auth, dav, OAuth consent) build decoded
strings and correctly keep :utf8. Pinned by a non-ASCII round-trip test.

If you authored non-ASCII content via the connector before this release, the
SOURCE on disk is fine - only the read-back display was wrong; re-reading after
deploy shows it correctly.

## Front-matter quotes

title: "Welcome" yielded a literal "Welcome", which a template then double-quoted
(the doubled review quotes). parse_yaml_front_matter now strips one matched pair
of surrounding quotes.

## Page-aware verbs (SM087, completed)

- create_page(slug, title, subtitle, body, register) - assembles the front
  matter; errors if the page exists.
- delete_page(slug) - removes the page + its .brief, reports remaining references
  (nav, other pages); registries auto-refresh.
- rename_page(old, new, update_links) - moves the page (carries .brief + ACL); with
  update_links, rewrites internal links to the old path across pages (best-effort;
  nav.conf not rewritten).
- write_file now validates on write (warnings/issues in the result).
