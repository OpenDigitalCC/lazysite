Release 0.4.18 - One-command manager bootstrap

Makes "getting started" a single built-in command, and tidies the dev server's
signal handling. Full suite green; perlcritic clean; strict SBOM gate green;
secrets clean.

## One-command manager bootstrap (SM093)

Making the manager usable on a fresh install used to take four fiddly, path- and
permission-sensitive steps. Now it is one idempotent command:

  perl tools/lazysite-users.pl --docroot DOC setup-manager [PASSWORD]

It creates the manager account (or sets its password), generates and prints a
strong password if none is given, creates the admin group with the user in it, and
ensures `manager: enabled` + `manager_groups` in lazysite.conf - never duplicating
or overriding an operator's existing values.

The Hestia deploy (`lazysite-hestia-deploy.sh`) now runs it automatically on a
fresh install, as the domain user after the chown, so a brand-new site is
manager-ready from the single deploy command - the printed summary gives the URL,
username, and generated password. No follow-up password/group/conf steps, and no
"Permission denied" trap from a root-owned tree (the deploy chowns first; running
install.pl directly as root was the unsupported path that produced it).

## Dev server signal cleanup (SM091)

The dev server traps SIGINT/SIGTERM and exits cleanly so its END block runs,
removing the temporary browse cache and error file. Previously Ctrl-C / kill left
the /tmp browse cache behind.
