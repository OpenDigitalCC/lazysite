Release 0.4.49 - Editor robustness + back button; stats log auto-detect

Two operator-reported fixes. Full suite green; perlcritic clean; strict SBOM gate green.

## Editor: always shows the edit box, and a way back

The file editor is a full-screen overlay (it covers the left sidebar by design). Two
problems: a hiccup in post-load setup could leave that overlay covering the page with no
visible edit box, and there was no way back to the menu once inside.

- loadFile() now builds the editor first, before the breadcrumb/download wiring, so the
  edit box renders even if a later step fails.
- The toolbar gains a "<- Files" back link, and Esc returns to Files when there are no
  unsaved changes (a dirty editor stays put).

## Visitor Stats: domain-qualified access-log auto-detect

Auto-detect now checks common access-log locations (Hestia domain logs,
/var/log/apache2, /var/log/nginx, ...) but only matches a log QUALIFIED BY THIS SITE'S
DOMAIN - taken from lazysite.conf site_url, else the Hestia docroot layout. So a shared
log directory never yields another site's log. The first readable match is used and
persisted to stats.conf (autoconfig); if nothing matches, the result asks the operator
to set the path rather than guessing.
