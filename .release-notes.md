Release 0.4.31 - Manager UI: no more native dialogs (SM109 phase 2b)

Completes the dialog modernization started in 2a. Full suite green; perlcritic clean;
strict SBOM gate green.

## What changed

The last 17 browser-native confirm()/prompt()/alert() call sites are converted to the
promise-based mgConfirm/mgPrompt modal:

- files: rename, new file, new folder (prompt); delete-selected, upload-overwrite
  (confirm)
- nav: edit item (two chained prompts), remove item (confirm)
- themes: activate, deactivate, delete, install-release (confirm); rename (prompt)
- plugins: per-action confirm (conditional) + delete handler
- cache: clear all
- edit: take-over-lock (confirm), and its stray alert() now a toast

There is no browser-native dialog left anywhere in the manager - every confirm and
prompt is the styled, keyboard-friendly modal, with red danger styling on destructive
actions.

## SM109 status

Phases 1 (token reskin), 2a (switches + toasts + Users modals), 3 (shared style
system) and now 2b (all dialogs) are shipped. Remaining: phase 4-5 (grouped side-nav
+ command palette) and phase 6 (dark mode - now a single token block).
