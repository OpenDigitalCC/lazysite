Release 0.5.15 - Channel capabilities + permission viewer (groups groundwork)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

- Capabilities are now split into Channels (ui / webdav / api / mcp - WHERE you may
  operate) and Actions (content, themes, layouts, config, users & groups,
  analytics, ... - WHAT you may do). You need both. Edit them per group on the
  Groups page.
- Six default role groups: content-editors, design-team (layouts & themes),
  agent-ai, mcp-ai, user-managers, plus lazysite-admins (everything).
- Each user card has a read-only "Permissions (derived)" grid - channels across,
  capabilities down - with each granted cell tooltipped by the group(s) granting
  it. View-only; visible to user managers.

Additive: nothing new is enforced yet. The next release flips to groups-only
resolution and removes the per-user toggles.
