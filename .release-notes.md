Release 0.4.32 - Manager UI: side-nav, command palette, dark mode (SM109 phases 4-6)

The finale of the manager-UI modernization. Full suite green; perlcritic clean;
strict SBOM gate green; TT template renders verified.

## Grouped side navigation

The nine-item top bar is replaced by a grouped left sidebar in a flex shell:

- Content: Files, Navigation, Themes, Plugins
- Access: Users
- System: Config, Cache, Backups, Audit

The header keeps the brand and tools and gains a palette trigger. The active page is
tinted with the accent; on narrow screens the sidebar collapses to a wrapped row.

## Command palette

Ctrl/Cmd-K (or the header button) opens a palette to jump to any manager page or run
a command - View site, Toggle dark mode, Sign out. Type to filter, arrow keys to move,
Enter to go, Escape to close. The keyboard-first entry point the Linear/Stripe-class
tools set as the expectation.

## Dark mode

A single [data-theme="dark"] block reassigns the surface / border / text / accent /
status / shadow / header tokens, and every component inherits it for free (the payoff
of the token system built in phases 1 and 3). A header toggle flips and persists the
choice to localStorage; the initial theme is applied before first paint from the saved
choice or the OS prefers-color-scheme, so there is no flash of the wrong theme.

## SM109 complete

Phases 1 (token reskin), 2a (switches + toasts + modals), 2b (all dialogs), 3 (shared
style system), and 4-6 (side-nav, palette, dark mode) are all shipped. The manager has
gone from a flat Bootstrap-default surface with native dialogs to a token-driven,
keyboard-navigable, dark-mode-capable console.
