Release 0.4.75 - Cache-safe sign in/out for themes

Full suite green (1832); perlcritic clean; strict SBOM gate green.

Themes that rendered the sign in/out control with a server-side [% IF authenticated %]
showed "Sign in" while logged in - pages are cached and shared, so the anonymous
state was baked into the cached HTML. New [% auth_control %] variable gives a theme
a ready-made control: a Sign in and a Sign out link, both hidden, that the injected
auth-sync script reveals from the lzs_session marker cookie client-side - correct
whether the page is cached or fresh. Themes drop it into the header (styled via
.ls-auth* classes); the layout-authoring docs document the pattern.
