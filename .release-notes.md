The application-side batch from the eight-dimension review (the operational
items are held for pre-launch, per docs/review/2026-07-01-eight-dimension/):

- Capability lookups now go through ONE shared helper (with the processor's
  module-free copy formally recorded in the project's first ADR). Along the
  way this fixed a real latent bug: a non-ASCII group description or user
  email silently emptied the whole settings read on the canonical path.
- The release is now gated end to end: compile sweep, security-themed lint,
  benchmark regression (tightened to 2x with recorded host provenance) and
  the instrumented coverage floors (statements AND branches) all refuse a
  bad build inside tools/release.sh.
- Coverage measurement is finally honest: lazysite-auth.pl and the
  Plugins/Upload handler modules were never "uncovered" - the tests dropped
  PERL5OPT when rebuilding %ENV, so the instrumentation never reached the
  child processes. Fixed; auth.pl now sits in the gate at 82%/61%.
- The benchmark now times the real render (84 ms) separately from the
  cache-hit serve (60 ms) instead of conflating them.
- payment-demo self-describes like every other plugin (it appears in the
  plugin list marked DEMO ONLY); the flaky login rate-limit test is de-flaked.

Note for operators: release runs now include the instrumented coverage pass,
which adds ~10-15 minutes to tools/release.sh. That is the intended cost of
making a coverage regression unshippable at the Commercial regime.
