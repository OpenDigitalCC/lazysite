Release 0.4.1 - Files-UI overhaul (SM077) + field-report/review fixes (SM080/081/078)

This release builds on the 0.4.0 modular-refactor milestone with one feature and
three fixes, all on the `claude/hestia-install-fixes` branch. Full suite 1454
green; perlcritic clean; strict SBOM gate green (180 components); secrets gate
clean. `dist/lazysite-0.4.1.tar.gz` built (150 files); 0.4.0 tarball removed.

## Feature - SM077 Files-manager UI overhaul

- **Editable permissions** - the owner chip on each file row expands in place to
  inline Read/Write editors (users and `@groups`); Save -> `acl-set`, clearing
  both -> `acl-remove`. Owner defaults to the caller for a new ACL, preserved
  otherwise.
- **Inline rename/move** - new cookie-only `move` action: validates + deny-checks
  both ends, refuses an existing target or a live foreign lock, enforces
  write-ACL on the source, moves the file + `.brief` sidecar + generated cache,
  and re-keys the ACL.
- **Lock indicator** - a glyph for a file held by another session / WebDAV.
- **`@group` ACLs** - `Auth::Acl` matches a `@group` entry against the requester's
  X-Remote-Groups; a token/WebDAV partner carries none, so `@group` never matches
  it (safe default).
- `action_list` now returns each file's read/write lists + lock state.

## Fixes

- **SM080** - the theme-asset mirror is now rebuilt on every theme/layout
  activation (not only on a repo install), so `theme_assets` resolves for a
  copied-then-activated layout (copy-then-activate is zero-edit). Partner
  briefings reconciled with the dhcf/Barn field reports.
- **SM081** - `form-targets` read parses mixed `handler:`/`type:` configs in
  document order (it used to drop the `type:` targets when any handler existed).
- **SM078** - the audit trail records the target of each action (path / config
  key), with a backward-compatible reader and a Target column in the Audit page.
