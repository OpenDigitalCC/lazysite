Release 0.5.29 - Synthesised error surface; raw log download removed

Full suite green (1998 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

The Visitor Stats page could reveal underlying log data. Two changes close that:

- "Recent server errors" is now synthesised, like the visitor classes. Each recent
  error-log line is reduced to its Apache error code (or module) with a plain-English
  category and a count - e.g. "Probe for a non-existent script (scanner noise): 22",
  "Request denied by server configuration: 204". No raw lines, client IP addresses,
  referer URLs or file paths are shown. (Verified against a real 8,000-line error log.)
- The raw access-log download is gone: the download link, the action behind it, and
  the plugin option that offered it are all removed, so the underlying access log can
  no longer be pulled through the manager.

Note on the sample error log you shared: the recurring "End of script output before
headers: lazysite-auth.pl" entries are auth.pl exiting without emitting headers -
consistent with the pre-upgrade issues you mentioned. They now show only as a
"CGI script produced no headers (script error)" count, with nothing identifying.
