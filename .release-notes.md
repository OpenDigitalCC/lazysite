## Browse any tree of Markdown (SM091)

  perl tools/lazysite-server.pl --docroot /path/to/tree --auto-index

The dev server already defaulted to no-cache and the built-in fallback layout (no
theme). --auto-index adds the last piece for browsing an arbitrary tree:

- a directory with no index.md gets a generated listing - sub-folders and pages as
  links, labels from each note's front-matter title, the directory's README linked
  as an overview - instead of a 404;
- every rendered page gets a breadcrumb nav (index / area / page) injected at the
  top, so notes link back up without a nav.conf;
- it writes NOTHING into the tree: scaffolding seeding is suppressed, and the
  processor's compile/layout cache is relocated to /tmp via the new
  LAZYSITE_CACHE_DIR processor env (inert unless set - production and tests
  unchanged).

The server also stops seeding scaffolding into a non-lazysite docroot (it only
seeds a real lazysite site - one with lazysite/ or lazysite.conf.example); --no-seed
forces that off anywhere.

Documented in the README quick-start ("Browse any folder of Markdown"), the
dev-server section of starter/docs/development.md, and --help.

## Security: production never lists a directory

Auto-index is a local-dev-only opt-in, off by default. A full install never reveals
a file list, on three layers: the processor returns 404 for a directory with no
index.md (no listing code at all); the Apache config ships Options -Indexes (Hestia
vhosts + the generic install doc); and FallbackResource only routes non-existent
paths, so a real directory with no index.html returns 403, not a listing. Locked by
t/unit/processor/23-no-directory-listing.t.
