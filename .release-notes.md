Release 0.5.26 - Migration: show legacy static HTML until Markdown lands

Full suite green (1992 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

Migrating an existing static site to lazysite used to hide the old pages: as soon
as the domain was switched to lazysite-app, every clean URL 404'd because the
engine only served Markdown (.md) or remote (.url) sources. Now a clean URL with
no Markdown source but a matching static file is served, so the site stays visible
while an agent builds the theme and converts pages one at a time.

- The engine serves a sibling `<base>.html` verbatim when there is no `.md`/`.url`
  source (any front end, including the dev server).
- On Hestia (lazysite-app), a rewrite serves the static sibling directly, choosing
  `.shtml` first so Apache still expands Server-Side Includes - your SSI header and
  footer keep working during the migration - and `.html` otherwise.
- A Markdown page always wins: the moment `about.md` exists, `/about` renders it,
  no switch to flip. Run a cache-invalidate at the end to clear any stale render.

Note: the rendered-page cache lives at `<base>.html`, the same name as a legacy
static page, so once a page is converted its `.html` becomes the cache. Keep your
originals in a separate copy if you need to refer back to them.

The vhost rewrite is new Apache config; it mirrors the existing homepage rule and
is self-limiting, but validate it on a rebuilt domain in your environment.
