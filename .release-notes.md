## Visitor Stats (SM083)

A read-only, opt-in plugin (plugins/stats.pl) that turns the web server access log into
on-site visitor statistics - a modern awstats/webalizer. It deliberately complements the
audit trail: the audit log records material actions (auth, edits, deletes), NOT browsing;
the access log captures every visit, including cache hits served directly by the web
server, so it is the right source for analytics.

What it reports, over a configurable window (default 30 days):

- Hits, unique visitors, and data served.
- Top pages and top referrers.
- Status-code breakdown and per-day hit counts.

Privacy and noise controls:

- Visitor IPs are anonymised by default (last octet zeroed); toggle off if needed.
- Bots and crawlers are filtered by default; toggle off to include them.

The access-log path is configurable on the Plugins page, or auto-detected (on Hestia,
../logs/<domain>.log). If the log is missing or the CGI user cannot read it, the page
shows a clear message rather than failing. A 10M-line guard bounds the scan; only
aggregates are held in memory.

The manager Stats page (in the sidebar and the Ctrl-K palette) runs the plugin and
renders summary tiles, a per-day bar chart, top-pages/referrers tables and the status
breakdown. Enable the Visitor Stats plugin on the Configuration page to switch it on.

## Future work

Incremental offset caching (so very large logs are not re-read each refresh) and
optional GeoIP are noted for later; the current scan is bounded and fast for typical
rotated domain logs.
