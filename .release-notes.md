Release 0.4.54 - Cache-buster on manager assets

The conclusion of the "editor blank" investigation. Full suite green; perlcritic clean;
strict SBOM gate green.

## What was happening

On a deployed site, the manager assets were already current ON DISK - manager.css 55844
bytes, codemirror.min.js 170535 - but a caching layer in front of Apache (nginx's
open_file_cache, or a CDN) kept serving the older, smaller copies. So the editor's
CodeMirror CSS/JS were stale and it mounted invisibly, even though the layout (served
fresh by Apache) was correct.

## The fix

Every manager asset URL - the manager.css <link> and all the CodeMirror <script>/<link>
tags - now carries ?v=<version>. When a release changes the version, the URL changes, so
any URL-keyed cache (the browser, a CDN) fetches the new asset instead of a stale one.
The version is exposed to every page via the renderer (from the install-state version).

nginx's path-keyed open_file_cache still needs a one-time reload to drop a stale entry
(systemctl reload nginx), but the cache-buster covers browser and CDN caches and prevents
this recurring on future upgrades.
