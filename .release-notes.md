Release 0.4.36 - WCAG contrast pass; setup-manager URL; README first-run

A contrast/accessibility pass (the recurring readability concern), a setup-manager
output fix, and README first-run guidance. Full suite green; perlcritic clean; strict
SBOM gate green.

## Colour contrast to WCAG

Contrast is a recurring concern, so it was done by the numbers: the main --mg-* text
tokens were audited against WCAG in both themes. One real failure surfaced -
--mg-text-light was #a8a29e in light mode (2.5:1, below AA) and #78716c in dark
(~3:1). That was the "feint / hard to read" text. It is now #79736d (light, 4.7:1) and
#9a948c (dark, 5.0:1), both AA. Everything else already met AA or better.

The dark CodeMirror editor palette is brighter: every syntax token (headings, bold,
links, strings, em, variables) now clears AAA (>=7:1), and the deliberately dimmed
comment/markers clear AA; the red and orange are lighter and easier to read.

The standard - WCAG targets, the audit table, and how to choose new colours - is
recorded in docs/reference/manager-colour-contrast.md.

## setup-manager URL

The "Manager ready" output printed the literal ${REQUEST_SCHEME}://${SERVER_NAME}
when run on the command line (those placeholders only resolve in the CGI environment).
It now expands what it can and falls back to a relative /manager/ when no host is
available.

## README first-run

A "First sign-in" section: open /manager directly (some hosts, HestiaCP included, seed
an index.html that the web server serves ahead of the lazysite homepage; /manager
always works), and get the first manager password with setup-manager.
