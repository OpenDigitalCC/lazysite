Release 0.4.30 - Manager UI: shared style system (SM109 phase 3)

Consolidates the manager's per-page inline styles onto the shared token system.
Visual change is intentional and small (more consistent surfaces, fixed white
backgrounds, tidier tags); no app logic touched. Full suite green; perlcritic clean;
strict SBOM gate green.

## What changed

The manager pages had reinvented components in inline <style> blocks (users.md alone
had ~57 lines), with hard-coded colours and a token bug Phase 1 surfaced. These are
now in manager.css:

- users.md component block (.mg-acc / .mg-box / .mg-line / .mg-tag / .mg-chk /
  .mg-inp / .mg-cred-*, etc.) moved to the shared sheet, token-driven. White surfaces
  now use --mg-surface (they had used --mg-bg, which Phase 1 changed to #fafafa);
  muted backgrounds use --mg-surface-alt. Status/capability tags gained tinted-pill
  backgrounds; .mg-inp gained a proper border + focus ring matching the form inputs.
- audit.md .audit-table (+ fail/info/detail) tokenised - removes hard-coded
  #c33 / #666 / #eee.
- config.md .mg-plugin-row and friends tokenised - removes #e5e5e5 / #fafafa.

edit.md reuses several mg-* classes, so it now inherits the consistent shared styling
without changes; its editor-specific block stays page-local for now.

## Why

Consistency is the first thing the eye reads as "slick", and it was leaking across
four different table treatments and a dozen ad-hoc colours. One token-driven system
also makes the later dark-mode phase a single block.
