Release 0.4.85 - Batch update reports channel-skipped sites

Full suite green; perlcritic clean; strict SBOM gate green.

The batch updater (lazysite-hestia-update-all) previously counted a stable site
that skipped an edge release as "updated". Now a channel skip is reported
distinctly: the deploy exits 4 on skip, and update-all reports
"Updated X, Skipped Y (stable channel), Failed Z" plus a final per-site summary
(domain, current version, update_channel, status). update_channel is also settable
via the control-API config-set path (validated all|stable).
