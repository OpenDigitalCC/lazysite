Release 0.4.60 - Redeploy a changed layout

Lets an agent push a layout fix to a site, not just install new layouts. Full
suite green (1754); perlcritic clean; strict SBOM gate green.

## What's new

- layout-install (control API) and the MCP install_layout tool accept update:true:
  when the named layout is already installed but differs, the old one is
  snapshotted and its layout.tt/layout.json overwritten, leaving its themes/
  intact. Without the flag a differing layout is left untouched (clearer error).
- Pairs with the lazysite-layouts pulse/chroma fix: their runtime JS palette now
  reads the active theme's CSS variables, so theme variants recolour the canvas /
  preset palette - push the updated layouts with install_layout(update:true).
