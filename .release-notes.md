Release 0.5.2 - Forms fail loudly when nothing accepts the submission

Full suite green (1880); perlcritic clean; strict SBOM gate green.

A form whose only delivery target was disabled (e.g. the delivery plugin turned
off) used to show a "thank you" while saving nothing. Now each delivery reports
success/failure and the handler errors ("This form is not accepting submissions
right now") - and logs it - when NO target actually accepted the submission.
