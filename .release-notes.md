Release 0.5.8 - Layout switch + backup proliferation fixes

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

- A layout can now be switched even when no theme of the same name exists: it
  falls back to the new layout's default theme rather than refusing with "Theme
  'X' is not declared for layout 'Y'". Explicitly naming an incompatible theme is
  still refused.
- Backup snapshots no longer chain their suffix
  (default-backup-T1-backup-T2-backup-T3...); a backup of a backup groups under
  the base name.
- Deleting a backup no longer snapshots it first, so backups can actually be
  cleaned up (the list shrinks).

Next (UI, queued): collapse backups into one panel per artifact (count + batch
delete), and consolidate the three theme/layout switchers into a single
"Installed layouts & themes" control.
