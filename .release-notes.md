Release 0.4.48 - Deploy hang fix; SSI overlay support

Two HestiaCP deployment fixes from live use. Full suite green; perlcritic clean; strict
SBOM gate green.

## Permissions sweep no longer hangs

The deploy's "permissions" step ran chown -R plus two per-file `find -exec chmod {} \;`
over the whole docroot. The runtime compiled-template cache (lazysite/cache/tt) mirrors
absolute paths in deeply-nested directories that the upgrade's .html-only cache clear
left behind, so the per-file sweep took many minutes and looked like a hang. It now
prunes that cache (regenerated on demand), batches the chmod with `{} +`, and prints a
completion line so a slow step is visible.

## lazysite-app overlays static SSI sites non-destructively

Applying lazysite-app to an existing static SSI site made every page show lazysite: the
template never enabled SSI, and DirectoryIndex was only index.html/index.htm - so an
index.shtml homepage was not served, the request fell through to lazysite, and it
rendered its starter index.md over the site.

- DirectoryIndex now includes index.shtml, so an existing SSI homepage is served and
  lazysite's markdown fallback never shadows it.
- The docroot enables Options +Includes and AddOutputFilter INCLUDES .shtml so .shtml
  pages are SSI-processed (needs `a2enmod include` - documented in the runbook). Harmless
  for markdown-only sites. Applied to both the http and https templates.

PHP/dynamic overlay stays out of scope for this template (it has no PHP handler) - revert
such a domain to its original template; the full non-destructive overlay is SM084. The
runbook gains an overlay section (homepage rules, the caveat, and how to remove a
previously-rendered index.html).
