Release 0.4.69 - Groups/Sessions pages, nested components, stats errors

Full suite green (1812); perlcritic clean; strict SBOM gate green.

## Manager

- Groups and Sessions are now their own pages under the Access menu group
  (/manager/groups, /manager/sessions); the Users page keeps accounts + per-user
  group assignment.
- Appearance catalogue: an installed layout offers "Update" (not "Install" beside
  an "installed" badge); an installed theme shows just the badge.

## Content components

- Nested components (a component inside another's content) now render.
- Reserved built-ins (::: include / oembed / form) work inside a component body.
- Fixed flow-style YAML in front-matter sections: (an inline list could report
  the wrong item count); block and flow both parse now.

## Stats

- Optional error-log surface: a new error_log config key (blank = auto-detect) is
  tailed cheaply and the most recent server errors render under the dashboard.
  The error log's path is never exposed.

## Docs / backlog

- SM089 (3D layout) moved to lazysite-layouts; the raw wishlist expanded into
  discrete items; new backlog items (session details, stats performance +
  visualisations, theme_assets fallback, remote-layout components). Layout config
  doc rewritten for the manifest catalogue.

bump-version.pl promoted VERSION to 0.4.69.
