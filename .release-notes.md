## Install/permissions doctor (SM093)

  perl tools/lazysite-check.pl --docroot /path/to/public_html [--fix]

Verifies an install is in working order and catches the classes of breakage seen
in the field:

- nothing under lazysite/ is foreign-owned - the root-owned-tree trap that breaks
  the no-suexec www-data CGI (expected owner is derived from the docroot, so it
  flags a tree left owned by root after running install.pl directly as root);
- the dirs the CGI must write (cache, logs, manager/locks, auth, forms,
  lazysite-assets) are group-writable + setgid;
- secrets (.secret, oauth.json, user-settings.json) are not world-accessible and
  the user store is not world-writable;
- the cgi-bin scripts and lazysite.conf are present/executable/readable;
- the manager is bootstrapped (admin group + a manager user with a password).

It reports OK / WARN / FAIL per check with a remediation hint and exits non-zero on
any FAIL; `--fix` applies the chmod (and, run as root, chown) repairs. The Hestia
deploy (lazysite-hestia-deploy.sh) now runs it as a final verification step, so the
root-owned-tree trap is caught - and fixable - at deploy time.

This release supersedes 0.4.18 for a rollout: it carries both the one-command
manager bootstrap (setup-manager, auto-run by the deploy) and the doctor.
