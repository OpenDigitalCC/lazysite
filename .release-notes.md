Release 0.4.28 - Manager UI reskin, phase 1

The first, lowest-risk slice of the manager UI modernization (SM109): a stylesheet
reskin, no application logic touched. Full suite green; perlcritic clean; strict SBOM
gate green.

## What changed

Because every manager page already consumes the `--mg-*` CSS variables, retheming
manager.css reskins the whole manager in one edit:

- A calmer token set: warmer neutral surfaces, a single confident indigo accent
  (distinct from the old Bootstrap blue), desaturated status colours, softer radii
  (6/10px), a real elevation scale (shadow sm/md/lg), a focus ring, and a disciplined
  type scale (15px base, one 13px small size, 18/24px titles).
- Cards gain soft elevation and a touch more padding; inputs and buttons share the
  new focus ring; the remaining hard-coded Bootstrap-blue/grey values are gone.
- Buttons: the primary button is solid accent with weight and a hairline shadow, and
  "one primary button per view" is applied to the main actions that were ghost/outline
  (Users: Add user, Add group; Config: Save).

## Why this first

It is the biggest visible lift for the least risk: one stylesheet, no behaviour
change, instantly visible across every page on the next deploy. The remaining phases
in SM109 (toggle switches for capabilities, a unified toast + confirm-modal to retire
the native confirm()/prompt(), a grouped side-nav + command palette, and dark mode)
are larger and are sequenced there.
