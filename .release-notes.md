Release 0.4.24 - Users-page UX; fleet health summary

Manager Users-page clarity + a consolidated fleet health summary. Full suite green;
perlcritic clean; strict SBOM gate green; secrets clean.

## Users page (SM094)

- Sub-users nest under their parent account and collapse with it - the list expands
  as the account hierarchy it is.
- Capability toggles are hidden for a manager-group (operator) account - they are
  overridden by the manager role - with a note naming the group; whoami now returns
  manager_groups so the page can tell.
- The capability section is relabelled "Publishing access (WebDAV / control API / AI
  connector)" so it reads as the partner grant it is.
- For an AI / backend account, interactive-login credentials (password, setup link,
  2FA) are hidden; only the Token it actually uses remains.
- The Add-group "first member" is a dropdown of existing accounts.

## Fleet health summary

lazysite-hestia-update-all.sh now ends with a consolidated summary of the doctor's
warnings and failures grouped by site, so you can see outstanding items at a glance
instead of scrolling each site's block.

## Agent brief

The WebDAV/API onboarding brief now tells an implementation agent to use only that
path and not a co-discovered MCP connector for the same account - a real Claude Code
confusion where it detected both and looped on failing attempts.

## Backlog captured

SM100 (one Connect flow per account), SM101 (a "do not retry" signal for permanent
tool failures). The remaining Users-page item is SM100.

bump-version.pl promoted VERSION 0.4.23 -> 0.4.24 (NEXT_VERSION -> 0.4.25).
