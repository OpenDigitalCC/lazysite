Release 0.4.33 - Dark-mode contrast fixes; plugin toggle safety

Fixes the dark-mode contrast issues found by living with 0.4.32, plus a plugin-toggle
safety fix. Full suite green; perlcritic clean; strict SBOM gate green; TT render
verified.

## Dark-mode readability

The bulk of the "unreadable on dark / doesn't invert" reports had one root cause: page
styles referenced var(--mg-bg-alt, #f6f6f6) but --mg-bg-alt was never a defined token,
so they fell back to a hard-coded light colour and never inverted. Aliasing the legacy
names (--mg-bg-alt / -muted / -hover) to the real surface token repairs the Users
instruction pane, the Backups top panel, the file-row expand box, the onboarding cards
and the permission chips in dark mode at a stroke.

On top of that:
- Links are now tokenised to the accent in both modes (they were browser blue/purple -
  unreadable on dark, visited links worst). Audit links, the domain-note pane links
  and misc inline links all read now.
- Sidebar group titles are stronger than the items beneath them.
- Ghost buttons (download, add brief, row actions) were too feint -> full text colour.
- r/w access flags are bolder; permission chips, badge/editor greens, the footer and
  the onboarding code box are tokenised (were #fff / #eee / #999 / #1a7f37 / #c33).
- The file editor uses CodeMirror, whose bundled theme is light-only; it now has a
  [data-theme=dark] variant, so the "extra" and content editors invert.

## Plugin toggle safety

The plugin row was a <label>, so clicking anywhere on the bar toggled enable/disable -
easy to do by accident. It is now a <div>; only the switch toggles.
