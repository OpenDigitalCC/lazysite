Release 0.5.32 - Fix manager-bar links unclickable over themed headers

Full suite green (1998 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

When the manager bar was moved into normal flow (0.5.28), it lost its own stacking
layer. On a theme whose header is positioned with a z-index (like the explorer
theme's sticky, blurred header), that header painted over the bar where the two
meet and intercepted the clicks - so the Manage / Edit / Sign out links did nothing.

The bar now sits in a `position:relative` layer with a maximum z-index: it still
stays in the page flow and scrolls away with the page (no overlap), but it renders
on top of the theme's header, so its links are clickable again.
