Release 0.4.66 - Data-driven sections (content components complete)

Final phase of content components. Full suite green (1799); perlcritic clean;
strict SBOM gate green.

## What is new

- Front-matter "sections:" lets a page describe itself as a list of sections,
  each a single-key map (component name) of data. The layout iterates and
  dispatches each to its component as "data". A small self-contained nested-YAML
  parser handles the subset (sequences, nested maps, inline flow maps and seqs,
  quoted/bareword scalars) - no new dependency, and only pages that declare
  "sections:" are affected.
- With Phases 1-2 (INCLUDE_PATH + markdown filter, and fenced ::: components),
  content components are complete: authors write Markdown plus a little structured
  data; all HTML lives in layout-owned, theme-styled components.

Layout author note: dispatch with braces - INCLUDE "components/${type}.tt" - because
bare "$type.tt" is parsed by Template Toolkit as the dotted variable type.tt.
