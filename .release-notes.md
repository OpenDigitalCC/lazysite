Full suite green (2,066 tests, 143 files); perlcritic clean (severity 4 +

The second application-side batch from the eight-dimension review (the
operational items remain held for pre-launch, per
docs/review/2026-07-01-eight-dimension/):

- Writes fail closed. The processor cache write and the form-handler append
  now drop the tempfile and log the event on a short write or failed flush,
  instead of renaming a torn page into place. A new failure-injection test
  reproduces disk-full (via ulimit + an XFSZ-ignoring harness, no root) and
  concurrent-PUT races to pin the behaviour.
- Backups can be restored from the manager. The Appearance panel gains a
  Restore button; the restore takes a safety snapshot first, overlays the
  archive, and clears only cached pages that still have a Markdown source -
  legacy static pages are preserved. Full round-trip test added.
- The documentation is current and now has a threat model. New docs/SECURITY.md
  is a STRIDE assessment with OWASP ASVS L1 control mapping (the five priority
  entries: forged trust headers, hostile layout.tt, secrets under the docroot,
  partner write-boundary bypass, CGI DoS). The doc set is aligned with the
  capability model (manager access is the `ui` capability on a group), and six
  ADRs now record the standing architectural decisions.
- Housekeeping: docs/POLICY.md corrects the CRA citation to Reg. (EU) 2024/2847;
  reported-issues.md records RI-001 (the dev server drops the second Set-Cookie
  header - dev-server only; fix queued next).

Note for operators: this is a docs- and reliability-weighted release; no change
to the request/render path or the on-disk formats. The logrotate snippet under
installers/hestia/ is shipped but its deployment is deliberately held until
formal launch.
