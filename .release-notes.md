Release 0.4.3 - Complete audit trail, @group over WebDAV, Files rights editor

Correctness-focused follow-up to 0.4.2. Full suite 1479 green; perlcritic clean;
strict SBOM gate green (182 components); secrets gate clean.
dist/lazysite-0.4.3.tar.gz built (152 files); 0.4.2 tarball removed.

## Fix: the audit trail was incomplete

The trail only covered the manager control API, so a publishing partner's WebDAV
writes (added images, edited pages) and MCP tool calls were invisible. audit_log
is now a shared Lazysite::Audit module called by all writers; origin is recorded
as ui (cookie), api (token), dav (WebDAV) or mcp. WebDAV records the method +
path (+ destination for MOVE/COPY).

## Fix: @group ACLs were ignored over WebDAV

lazysite-dav.pl had a private acl_allows predating SM077 that did not understand
@group entries, so a group grant set in the manager UI was silently dropped over
WebDAV. It now delegates to the shared Lazysite::Auth::Acl (resolving the user's
groups from lazysite/auth/groups), so the manager, MCP and WebDAV enforce
identical rules.

## Feature: Files config card - unified rights editor

The per-file card replaces its two native multi-selects with one "People &
groups with access" list: each principal is a chip with r / w toggles and a
remove control, added via a typeahead. The audit History link moved into the
card; the card is roomier.

## Rollout

This release also carries the Hestia template-hook lib/ fix from 0.4.2. Roll it
to every lazysite site on a Hestia host, refreshing the shared template so the
corrected hook propagates:

    # on the Hestia host, as root, after extracting the tarball to /tmp:
    tar xzf lazysite-0.4.3.tar.gz -C /tmp
    /tmp/lazysite-0.4.3/installers/hestia/lazysite-hestia-update-all.sh \
        --templates /tmp/lazysite-0.4.3
