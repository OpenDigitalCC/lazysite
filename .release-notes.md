Release 0.4.76 - install --verify (deploy-gap detector)

Full suite green (1833); perlcritic clean; strict SBOM gate green.

The reported version comes from a side-car (.install-state.json); a partial deploy
that left a stale .pl could report the new version with old code. Now:

- install.pl --verify --docroot D --cgibin C re-hashes every code-bucket file at
  its installed destination against the shipped release-manifest.json; exit 0 if
  the running code IS this version, exit 1 (naming the files) if not.
- A normal install runs the same check and surfaces any mismatch in the summary.
- The Hestia deploy runs --verify after installing and fails the deploy on
  mismatch, so a partial deploy can't silently stamp the new version.
