Release 0.4.56 - Fix blank manager editor (auth-sync injection)

The real root cause of the blank editor - found once the browser console finally
showed "SyntaxError: literal not terminated before end of script". Full suite
green; perlcritic clean; strict SBOM gate green.

## What was wrong

The processor injects a tiny SM099 auth-sync <script> before </body> on every
page. It used a non-global s{</body>}{...}, matching the FIRST </body>. The editor
page builds an iframe preview with frame.srcdoc = '<html><body>...</body></html>'
- a literal </body> inside a JS string, and the first </body> in the document.
The auth-sync <script>...</script> was spliced in there, and its </script> closed
the editor's own inline <script> early. The whole editor script failed to parse,
CodeMirror never initialised, and the panes were blank - identically on every
site, with no stale-asset symptom and no other console error.

This is why earlier rounds (stale nginx assets, the ?v= cache-buster, the nginx
reload) never fixed it: those were real but separate; the editor was dead before
CodeMirror's bytes mattered.

## The fix

Anchor the injection to the document's real closing tag with a negative
lookahead: s{</body>(?![\s\S]*</body>)}{...}. A </body> inside page content or a
JS string is now left untouched. Regression test t/unit/processor/24 pins it.
