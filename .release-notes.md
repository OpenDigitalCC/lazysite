Release 0.5.22 - Retire the manager group: Manager-UI is the `ui` capability

Full suite green (1967 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

- Access to the Manager UI is now the `ui` channel capability, granted through a
  group - the same central resolver every other surface uses. A user in any group
  carrying `ui` lands in the manager on login and passes the manager gate, with no
  separate "manager group" concept to reason about.
- The unrestricted account-management operator (manage any account, bypass per-file
  ACLs) is now the `manage_users` action, carried by a group.
- The old `manager_groups` config still works as a fallback, so existing sites keep
  working unchanged - nothing is locked out by this release.
- The Groups page drops the transitional "Manager group" toggle; its note now
  points at the `ui` channel and the "Users & groups" action instead.
