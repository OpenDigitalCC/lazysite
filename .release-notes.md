Release 0.4.2 - MCP server v1, Files-UI v2, and the Hestia lib/ upgrade fix

Builds on 0.4.1 with one operator-critical fix, a new integration surface, and a
Files-page redesign. Full suite 1473 green; perlcritic clean; strict SBOM gate
green (181 components); secrets gate clean. dist/lazysite-0.4.2.tar.gz built
(151 files); 0.4.1 tarball removed.

## Operator-critical: Hestia 0.4.x upgrade fix

The lazysite-app template hook now pre-creates the site-root `lib/` (for the
SM079 modules) as root, alongside `plugins/` and `tools/` - the domain root is
mode 0551, so install.pl as the domain user cannot create it. Without this,
upgrading a domain whose root is not user-writable failed with
`mkdir .../lib: Permission denied`. install.pl also gives an actionable error
now. **Operators upgrading from 0.3.x/0.4.0/0.4.1 must refresh the lazysite-app
template** (`update-all --templates`) so the hook creates `lib/`.

## Feature: MCP server v1 (SM076)

`lazysite-mcp.pl` - a remote Streamable-HTTP JSON-RPC MCP server so an AI client
(Claude.ai custom connector, Claude Desktop/Code) can run site-maintenance tools:
whoami, list/read/write/move/delete files, set_permissions, activate_theme,
activate_layout. Static bearer auth (`<partner-id>:<lzs_ token>`) verified by the
same path as the control API, so capabilities + per-file ACLs bind identically.
OAuth + SSE + set_config are deferred. See the SM076 doc for connector setup.

## Feature: Files-manager UI v2 + richer audit (SM077)

The Files page is redesigned - icon + name on the left; an Access column
(owner + colour-coded r/w, g for a group), a Modified column (relative, absolute
on hover, linking to that file's audit history), right-side selection, and a
chevron opening a per-file config card (permissions multi-selects, Download,
Add/Edit brief, Move, Save). New `principals` action feeds the pickers. The audit
trail gains an origin column (ui/api) and a target filter.
