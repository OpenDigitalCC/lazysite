Release 0.4.83 - Form binary uploads

Full suite green (1859); perlcritic clean; strict SBOM gate green.

Forms can now accept binary uploads (images, PDFs, ...):

- Author a file field: `photo | Photo | file accept:image/* required` (add
  `multiple` for several). The form auto-switches to multipart/form-data.
- Per-form limits in FORMNAME.conf: upload_max_files, upload_max_kb (per file),
  upload_accept (extensions). A violation is rejected up front with a specific
  message to the visitor.
- Files are stored in a per-submission subdir next to FORMNAME.jsonl
  (FORMNAME.files/<id>/<name>); the submission JSON records _files + _files_dir,
  never the bytes. Filenames are sanitised (no path traversal).

See features/authoring/forms.md and ai-briefing-configuration.md.
