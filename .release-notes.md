Release 0.4.39 - Audit filter dropdowns; read-only data files

Two manager improvements plus a batch of captured theming feedback. Full suite green;
perlcritic clean; strict SBOM gate green.

## Audit filters as value dropdowns (SM119)

The Audit page's User and Target filters are now dropdowns populated from the distinct
values that actually appear in the log (action_audit returns the facets), each with an
"(all)" default and a "(none)" option for blank-valued entries (public form
submissions, install/denied events). Pick from what is really there instead of typing.

## Read-only data files in the editor (SM115)

Append-only data - form submissions and .jsonl files - now opens read-only by default
with an explicit "Edit anyway", since editing the whole file would clobber records
appended concurrently. This is belt-and-braces over the editor's existing mtime
conflict guard, which already refuses a save when the file changed underneath. The
"View submissions" link already existed on the plugins page.

## Captured: theming experience report (SM120-124)

A detailed field report on building/publishing themes was captured as feature-requests:
SM120 (preview-only per-page theme: override - the single highest-leverage item),
SM121 (wire nginx /dav/ on WebDAV enable + a 401 health check), SM122 (config
self-service for manage_config tokens), SM123 (list installed themes + asset-mirror
lifecycle), SM124 (connector onboarding alignment - brief = whoami, grant matches task,
admin-group vs token-capability planes, MKCOL idempotency).
