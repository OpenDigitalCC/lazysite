Release 0.4.50 - Editor always shows the edit box

A follow-up to the editor report ("still empty; the page ends at 'extra'"). Full suite
green; perlcritic clean; strict SBOM gate green.

## What was happening

Rendering the editor page through the processor confirmed the HTML and the <script>
survive markdown intact and query.path is wired correctly - so a blank editor meant
loadFile() returned at its empty-path guard BEFORE building the editors, leaving the
full-screen editor overlay covering the page with empty CodeMirror divs (hence "the
page ends at extra", no edit box, no preview, and no JS error).

## Fix

loadFile() now calls initEditors() unconditionally and first, so the edit box always
renders. With no file path it shows a "No file selected" placeholder (and hides
Save/front-matter); with a path it proceeds as before. Combined with 0.4.49's
init-first change and the back link, the editor can no longer present an empty overlay.

If a real file still opens empty, the URL will lack ?path= - that points at how the
editor was opened (a Files-page link) rather than the editor itself.
