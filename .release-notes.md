Release 0.4.82 - Update channel (stable vs edge)

Full suite green (1840); perlcritic clean; strict SBOM gate green.

A site can choose which upgrades it accepts, so an unstable build can be kept off
stable customer sites. Not a self-updater - upgrades are still operator-deployed;
the channel is a site preference the installer enforces.

- Releases carry a channel: stable (release.sh --final) or edge (default).
- Sites set update_channel = all (default) or stable in Site settings.
- A stable site offered an edge upgrade SKIPS it: no files change, the installer
  exits 3 (clean no-op), and an 'upgrade-skipped' entry is written to the audit
  log. Fresh installs/reinstalls are never gated. The Hestia deploy treats the
  skip cleanly.

See features/configuration/update-channel.md.
