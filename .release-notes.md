Release 0.4.25 - One Connect flow; sub-user count badge

Finishes the Users-page UX: one connector-credential flow, and small hierarchy
cues. Full suite green; perlcritic clean; strict SBOM gate green; secrets clean.

## One Connect flow (SM100)

A partner account had three parallel connector-credential controls, each right for a
different client - and picking the wrong one failed confusingly (the live report:
the Claude Code token did nothing for Claude.ai, which is OAuth-only). They are now
one "Connect an AI assistant" section: pick the client and get the credential that
works, with the reason inline.

- Claude.ai / ChatGPT (web) -> OAuth connect code
- Claude Desktop (connector) -> token
- Claude Code / a script -> single-use pairing brief

The Credentials section is now interactive-login only (password / setup link / 2FA),
shown for human accounts; the token moved into the Connect flow.

## Hierarchy cues

- A parent account's heading shows "(+N)" sub-users.
- The Add-user parent default is reworded to "(top-level account - no parent;
  managed by you)" - the old wording implied it nested under a manager, which it does
  not.
