Lets lazysite be installed over a live HTML/SSI site and migrated page by page,
without losing existing content, plus recoverable content backups surfaced in the
manager. Full suite 1572 green; perlcritic clean; strict SBOM gate green; secrets
clean.

## Non-destructive install (overlay)

lazysite can now be installed over an existing static site without losing the
homepage. The installer clears index.html ONLY when index.md already existed (so
it was the cache rendered from it) - a freshly-seeded index.md or a static-site
overlay leaves an existing index.html untouched, and deploy.sh no longer deletes
it (it just warns if a placeholder shadows a new index.md). This is a narrowed
delete, not an opt-in mode that could be forgotten. Existing .html/.shtml/SSI
pages keep serving until a .md replaces them - migrate one page at a time.

## Content backups

Tarball snapshots of the site content (excluding the lazysite/ infra, so no
secrets) under lazysite/backups/, which is never served.

- A one-time pre-install snapshot (preinstall-<stamp>.tar.gz) is taken by the
  Hestia hook the first time lazysite is installed over existing content - the
  migration safety net.
- New manager Backups page (nav link) lists snapshots, takes manual ones on
  demand, and downloads them. Manager-only (not exposed to token clients);
  download validates a strict basename (no path traversal).

## Migration workflow

1. Back up first (automatic pre-install; or Backups -> Create backup now).
2. Install lazysite (overlay) - existing pages keep serving.
3. Migrate a page: add <page>.md; it renders over the old <page>.html; verify.
4. Repeat. The homepage migrates last - add index.md and remove the old
   index.html deliberately (never the installer).
