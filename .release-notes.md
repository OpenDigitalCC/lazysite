Release 0.4.6 - Claude.ai connector onboarding + injection-resistant briefs

Onboarding + partner-safety release on top of 0.4.5. Full suite 1482 green;
perlcritic clean; strict SBOM gate green (183); secrets clean.
dist/lazysite-0.4.6.tar.gz built (153 files); 0.4.5 tarball removed.

## Feature: one-click Claude.ai connector setup (SM076)

Two onboarding buttons on the Users page, matched to the audience:
- "Set up Claude.ai" (new) - mints a credential for the MCP connector's settings
  (never chat) and steps the operator through adding the connector, confirming
  with whoami, and a no-secret task prompt. The robust path for the web app and
  ongoing tweaks.
- "Generate agent brief" - the pairing-key + API/WebDAV flow for Claude Code or
  a script (key delivered out of band).

The MCP server itself shipped in 0.4.3, so the connector already works against a
deployed site; this makes the operator setup one click.

## Security: injection-resistant onboarding briefs

After a Claude.ai partner correctly declined a brief that embedded a secret,
asked an assistant to autonomously handle credentials, and read like a prompt
injection, the generated brief is reframed as operator-issued data to verify
(not commands), with explicit secret-handling guidance. The Users panel warns
the same and gained a Close button + supersede note.

## Docs: partner onboarding

An egress preflight ("confirm you can reach the site" - detect a blocked egress
and report early) and an operator onboarding-brief template.
