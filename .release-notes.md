Release 0.5.6 - AI visitor-log analysis (Phase 1)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

An AI agent can now analyse this site's visitor trends - on the operator's
direction - without ever seeing the raw access log.

- New `analytics` capability (off by default; grant it per connector/user).
- `analyse_visitors` MCP tool returns a sanitised JSON: per-day totals, a
  people/AI-assistant/bot/noise breakdown, top pages, referrers, status codes and
  a capped recent event stream. Never the raw log, the log path, any filesystem
  path, or a visitor IP - IPs are anonymised; events carry only a non-reversible
  network-level token.
- Built from the access log via an incremental cache (per-day buckets + a
  processed offset, rotation-aware), so each call parses only new log lines.
- Briefing at /docs/ai-briefing-stats explains the fields, the traffic taxonomy,
  what can be reported, and what must never be claimed (no PII / identity).

Phase 2 (next): the audit trail as the same JSON, gated on the same capability,
with the in-page Audit view reading it.
