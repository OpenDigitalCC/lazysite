Release 0.4.61 - Manager UI consistency

A consistency pass over the manager. Full suite green (1754); perlcritic clean;
strict SBOM gate green.

## What changed

- Page-title framework: the manager layout renders one consistent heading from
  each page's front-matter title, so pages no longer each hand-roll (or omit) a
  title and can't drift. The full-bleed editor opts out.
- Naming standardised between the nav and the page in both the sidebar and the
  command palette: Config -> Site settings, Stats -> Visitor statistics, Audit ->
  Audit log; "Cache Manager" -> Cache; "User Management" -> Users.
- Audit page: a failure reason expands on its own full-width row below the entry
  (was cramped in the narrow Status cell); the From/To date range sits on its own
  filter line so it never splits across a wrap.
