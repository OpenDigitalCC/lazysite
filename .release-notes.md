Release 0.5.18 - Fix Users page failing to list users

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

The batched account load introduced in 0.5.16 could return a row with a null
username (the map's $_ clobbered by file reads inside effective_settings), which
made the Users page throw "can't access property localeCompare, a.user is null"
and fail to list anyone. Fixed by capturing the username before the settings
lookup, plus a resilient front-end. Deploy over 0.5.16/0.5.17.
