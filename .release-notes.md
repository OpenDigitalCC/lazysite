Release 0.5.23 - Live permissions grid, honest analytics, the building-sites briefing

Full suite green (1975 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

Permissions

- The Users-page permissions grid now re-checks every time you open it (so changes
  made on the Groups page show up), with a Recheck button to force a refresh.
- New command-line grid for debugging user access:
  `lazysite-users.pl --docroot ... permissions USERNAME` prints the channel x
  capability matrix resolved from the user's groups, plus which group grants each.
- Creating a user, the group picker now shows what each group is for.

Analytics

- The visitor classifier no longer counts automation as human. Headless browsers
  and the common headless-driver tools are detected, and the documented opt-out
  User-Agent `lazysite-agent/<partner-id>` (or legacy `claude-code-agent`) keeps an
  agent's own screenshot/QA traffic out of the human figure.
- Infrastructure fetches (favicon, robots.txt, sitemap, llms.txt, /.well-known/,
  feeds) are treated as noise rather than human page views.

AI agent pack

- A new briefing, "building sites" (/docs/ai-briefing-building-sites), explains the
  content/layout/theme separation and why ordinary pages must not use raw mode
  (`api: true` / `raw: true`) - the root cause of one-off monolith pages. It is now
  part of the pack that both MCP and WebDAV agents receive on connect. Ask existing
  agents to re-read the pack to pick it up.
