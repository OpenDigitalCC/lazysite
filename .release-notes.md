Release 0.4.14 - Multi-word select options + lock take-over

The key fix from the live Claude.ai validation: multi-word select form options now
work. Plus a take-over button for a stale editor lock, and file size in the Files
list. Full suite 1635 green; perlcritic clean; strict SBOM gate green; secrets
clean.

## Multi-word select options

select: options with spaces were truncated at the first space (e.g. the /enquire
dog field, /reviews rating + consent), and quoting didn't help - neither the
renderer nor the validator honoured it. The rule parser now treats select: as
taking the REST of the rule line, so:

    dog | Dog | required select:No,Yes - one small to medium-sized dog

renders both options whole - no quotes needed. Surrounding quotes are still
tolerated and stripped. The validator drops the select: clause before checking,
so option words are no longer flagged as unknown rules. Convention: put select:
last among a field's rules. Affected content (/enquire, /reviews) can now be
fixed by simply leaving the options unquoted.

## Take over a stale editor lock

A file shown as "Locked by ..." in the editor now has a Take over button that
clears the lock and re-acquires it. Previously a lock orphaned by an editor left
open across a restart disabled save until the 5-minute TTL expired. The server
still refuses a live WebDAV lock. (Locks also auto-expire after 300s, and the
operator can always rm the .lock under lazysite/manager/locks/.)

## File size in the Files list

The Files page Modified column now shows the file size after the date (files
only).
