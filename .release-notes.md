Release 0.4.21 - Audit logins; .url editable; doctor checks writable config

Adds login/logout to the audit trail, makes .url files editable, and closes the
permission gap behind "nav cannot be written". Full suite green; perlcritic clean;
strict SBOM gate green; secrets clean.

Commits since v0.4.20 (oldest first):
- 109bbef  feat(audit): record login/logout and material auth events in the audit trail
- 4e3a8f5  fix: .url files editable; doctor checks CGI-writable config files
- fcab4e6  feat(users): groups "first member" dropdown; capture Users-page + nav backlog

## Login/logout in the audit trail

The auth wrapper logged authentication only to the application log, so a login never
showed in the manager Audit viewer. It now also writes audit events for: login
success (incl. localhost no-password), login failure with a reason, logout,
claim-redeem, and token exchange/rotate. Origin 'ui' for browser auth, 'api' for the
credential exchange/rotate flows.

## .url files editable

A .url file is a single remote URL; it was treated as binary, so the editor refused
it. It now opens as text. (A follow-on, SM096, will add a one-click "migrate to
local .md" that captures the remote body.)

## Doctor: config files the CGI must write

After a deploy whose permission pass did not run (e.g. an upgrade that aborted on a
root-owned tree), nav.conf / lazysite.conf were left not group-writable, so the
manager could not save them. lazysite-check now checks nav.conf, lazysite.conf,
auth/users, auth/groups and auth/acls.json are group-writable by www-data, and --fix
adds group-write. Combined with 0.4.20's deploy ownership normalisation, a re-deploy
now leaves every CGI-writable file correct.

Re-deploy this version with the fleet update. With 0.4.20's pre-install chown plus
