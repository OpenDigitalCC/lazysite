Release 0.4.37 - Fix hidden capability toggles for operators; audit deploys; settings groups

Contains an important correction plus the next slice of polish. Full suite green;
perlcritic clean; strict SBOM gate green.

## Fix: operator accounts could not be granted token capabilities (SM094)

Hiding the per-account capability toggles for manager-group (operator) accounts was a
mistake. Operator status (manager-group membership) only bypasses capabilities on the
browser-cookie/UI path. An operator account that ALSO connects with a partner token is
still gated by those capability flags on the WebDAV / control-API / MCP path - so a
member of lazysite-admins whose connector needed manage_themes/manage_layouts could
not be granted them, because the toggles were hidden.

The toggles are now always shown and settable, with a note on operator accounts
explaining they govern the token/connector path (which the manager role does not lift).
If you hit this, you can also set it immediately from the CLI:

    perl tools/lazysite-users.pl --docroot <docroot> set <account> manage_themes on
    perl tools/lazysite-users.pl --docroot <docroot> set <account> manage_layouts on

It applies live on the next request - no new token needed.

## Deploys are audited (SM117)

install.pl now writes an audit event at the end of a successful install/upgrade
(installed / upgraded from -> to, origin install), so a deploy shows in the trail.

## File delete button; settings groups (SM114)

Site settings are grouped under Identity / Appearance / Content / Access headers, and
the Files breadcrumb root is now the same folder icon as the editor's.
