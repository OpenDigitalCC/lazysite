Release 0.5.13 - Group-based capabilities (Phase 2, Groups page)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

The Groups page is now where you configure roles:

- Per-group capability toggles (the same set as the Users page) plus a "Manager
  group" switch. Members inherit the union of their groups' permissions.
- Member-centric roster: each group shows who is in it, a type-to-search box to
  add a user, and a remove control per member - no more a checkbox for every
  account (which did not scale).
- Create a role group with no member required; delete a group. The last manager
  group cannot be deleted or unflagged (lockout guard).

Next: Phase 3, the clean cut - capabilities resolve from groups only and the
per-user capability toggles come off the Users page (replaced by group membership
+ a read-only effective-permissions summary).
