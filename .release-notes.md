Release 0.4.74 - Audit names the changed setting on plugin-save

Full suite green (1827); perlcritic clean; strict SBOM gate green.

A config save was audited as "plugin-save | lazysite" no matter what changed. The
audit target now appends the saved setting key(s), so it reads e.g.
"plugin-save | lazysite (site_name)" / "lazysite (webdav_enabled)". Keys only
(values may be secrets); whole-form saves show "N settings".
