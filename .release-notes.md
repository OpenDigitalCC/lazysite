Release 0.4.29 - Manager UI: switches, toasts, modal dialogs (SM109 phase 2a)

The second slice of the manager modernization. Full suite green; perlcritic clean;
strict SBOM gate green.

## Toggle switches

The capability and plugin on/off toggles now render as switches rather than bare
checkboxes - pure CSS on the existing .mg-chk checkboxes plus a .mg-toggle class for
the plugin list. The Users page's "wall of nine checkboxes" now reads as state at a
glance, matching the Claude/ChatGPT settings idiom.

## Toast notifications

A single global toast (bottom-right, auto-dismiss, success/error variants) replaces
the warning bar and the per-page status line. mgShowWarning and the pages' showStatus
both route to it, so all existing feedback becomes a toast with no call-site changes.

## Styled confirm / prompt modal

A promise-based mgConfirm / mgPrompt modal (added globally in the manager layout)
replaces the browser-native confirm()/prompt() - the single strongest "unpolished"
tell in the manager. All six Users-page dialogs (generate credential, disable, delete
user, disable 2FA, delete group, rotate auth secret) now use it, with danger styling
on the destructive ones.

## Still to come (phase 2b)

The remaining native dialogs on files / nav / themes / plugins / cache / edit (17
sites) convert to mgConfirm/mgPrompt next - mechanical, same pattern. Then phases 3-6
in SM109 (style consolidation, side-nav + command palette, dark mode).
