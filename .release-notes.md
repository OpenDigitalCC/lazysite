Release 0.5.20 - Clean cut: capabilities resolve from groups only (SM095 c1)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

The permissions overhaul completes its core: capabilities are now EXCLUSIVELY
group-based. Per-account capability grants are no longer honoured (no hidden
grants) and can no longer be set - the single source of truth is group membership.

- New partners/sub-users get their capabilities via a role group, not per-account
  defaults (a connector gets a role with its channels + content + themes).
- whoami reports the full capability set (channels ui/webdav/api/mcp, manage_users,
  analytics) - previously it omitted the newer ones.
- The Users page capability toggles are replaced by a read-only "Capabilities
  (from groups)" summary; manage capabilities on the Groups page.

ACTION on upgrade: any account whose access came from per-account capabilities
must be added to a group that grants them (lazysite-admins keeps full access).
The "who had what" guidance and retiring manager_groups land in the next release.
