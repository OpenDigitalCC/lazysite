Release 0.4.11 - Form field types + connector editing tools (SM087)

More form field types, and a set of safer/higher-level MCP tools driven by live
ChatGPT use (patch editing, search, validation, site audit), plus the
generated-index refresh fix and audit-log pagination. Full suite 1615 green;
perlcritic clean; strict SBOM gate green; secrets clean.

## Form field types

tel (default validation pattern), date, time, number (min:/max: value bounds),
url, password; pattern:REGEX for custom validation; placeholder: rule. Values
needing spaces are quoted: placeholder:"Your full name", pattern:"[0-9 +()-]{7,20}".

## Connector editing tools (SM087, Tier 1 + Tier 2)

- replace_text(path, old, new) - patch a file by exact text instead of rewriting
  the whole file; errors if the text is absent (no silent clobber).
- search_files(query) - content grep with snippets.
- page_status(path) - source exists, render pending (will re-render next visit),
  public URL - confirm an edit reaches visitors without a web fetch.
- read_page / list_pages - page-level view with parsed front matter.
- validate_page(path|content) - unterminated front matter, missing title, invalid
  form rules, and a PUBLIC-DATA warning (Wi-Fi passwords, postcodes/addresses,
  phone numbers) so private guest-instruction details aren't published by accident.
- audit_site - broken internal links, orphan pages, missing titles, stale
  generated HTML, duplicate content blocks (repeated reviews/testimonials).
- invalidate_cache - force a page (or all) to re-render.
- Error responses carry a machine-readable `kind` (blocked, not-found, permission,
  binary, too-large) so the assistant can tell causes apart.

## Generated indexes refresh on change

A content delete/save/move now invalidates sitemap.xml / llms.txt / feeds, so a
deleted page no longer lingers in them (previously a 4h TTL). Covers the manager
and MCP write paths.

## Audit log

Material-event log now paginates at 50 rows/page (Prev / "Page N of M" / Next).
The new read/diagnostic tools are not audited (reads); writes (replace_text -> edit,
invalidate_cache, etc.) are.
