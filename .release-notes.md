Release 0.5.35 - Content provenance stamp + "is it ours?" checker

Full suite green (2003 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

Following the homepage-overwrite incident, lazysite content is now self-identifying,
so it can be told apart from an operator's own content without depending on the
install state.

- Every shipped starter page (home, contact, login, the error pages, ...) carries a
  `provenance: lazysite-starter` marker in its front matter. It is invisible on the
  page and harmless; it just records "lazysite shipped this."
- Run the doctor to see the picture: `perl tools/lazysite-check.pl --docroot PATH`
  now reports content provenance - how many pages are lazysite's and unmodified,
  lazysite's but customised (edited from the shipped version), or entirely
  operator-authored, and lists the customised and operator pages.

This is an audit signal only. The upgrade never overwrites your content: the 0.5.33
preservation fix still stands; this just lets you (and tooling) answer "is this page
likely ours?".
