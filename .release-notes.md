Release 0.5.3 - Reject contentless form submissions

Full suite green; perlcritic clean; strict SBOM gate green.

A form POST with every visible field blank and no file (an automated/blank submit)
was saved as an empty record behind a "thank you". It now errors ("Please fill in
the form before submitting.") and saves nothing. Upload-only forms are unaffected
(a file counts as content). Pairs with the prior fix that errors when no delivery
target accepts the submission.
