Release 0.5.27 - Theme-adaptive login form

Full suite green (1992 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

Following the 0.5.25 fix (which stopped the login form's CSS being mangled), the
form still looked out of place on a custom theme because it used fixed colours. It
now takes its accent, border and surface colours - and the button colour - from the
active theme's design tokens, with the previous values as fallbacks. So the login
page adopts whatever theme is active, and is unchanged on the built-in fallback
layout. No per-theme work needed: any theme following the standard token names
(accent / border / surface / bg / text / heading) styles the login form for free.
