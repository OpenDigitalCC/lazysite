Release 0.4.23 - Audit static-bearer connect; deploy chowns all install targets

Records a Claude Code / static-bearer connector's connection in the audit trail, and
fixes the last deploy ownership gap. Full suite green; perlcritic clean; strict SBOM
gate green; secrets clean.

Commits since v0.4.22 (oldest first):
- fix: audit static-bearer connect (Claude Code); deploy chowns lib/plugins/tools

## Static-bearer connect now audited

The OAuth (web) flow audits the connection; the static lzs_ bearer (Claude Code /
Desktop / a script) did not. The MCP now audits a `connect` (origin mcp) on the
FIRST use of a credential since issuance - so an active connector records its
connection once, without an entry on every tool call. Existing connectors whose
credential is already "used" will log a `connect` the next time the credential is
regenerated/reconnected.

## Deploy chowns all install targets

The deploy normalised ownership of the docroot + cgi-bin before the user-run
install.pl, but not the sibling lib/ plugins/ tools/ trees install.pl also writes. A
site whose lib/ was left root-owned (original install.pl run as root) therefore still
failed the upgrade copying lib/Lazysite/Audit.pm. It now chowns docroot, cgi-bin,
lib, plugins and tools first - so a re-deploy of such a site finally completes.

## SM099 captured

The sign-in/out button is baked into the shared page cache, so it can show the wrong
state to a visitor; captured as a client-side fix (marker cookie + small script).
