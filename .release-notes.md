Release 0.4.22 - Audit the connector lifecycle; audit.log writability

Makes a Claude.ai/ChatGPT connector's full lifecycle visible in the audit trail, and
catches the permission gap that silently dropped all audits. Full suite green;
perlcritic clean; strict SBOM gate green; secrets clean.

Commits since v0.4.21 (oldest first):
- feat(audit): full OAuth/connector lifecycle; doctor checks audit.log writable

## Connector lifecycle in the audit trail

The OAuth flow audited only the access-token issue. Now the whole connection shows:
oauth-register (a client registered), oauth-authorize (the connect code was redeemed
- consent - plus the failed case), connect (access token issued), and oauth-refresh
(the connector renewed its token). All origin=mcp. Tool writes were already audited;
reads (whoami/list/read) remain unaudited by design (material events only - read
visibility belongs to the access log / stats plugin, SM083).

## Why "nothing showed in the audit log"

Two reasons, both now addressed: most of the OAuth events were not being recorded
(added above); and, more fundamentally, an audit.log left not group-writable by the
www-data CGI makes every append silently fail - so NOTHING is audited (not logins,
not connects). lazysite-check now checks audit.log writability and --fix repairs it.
On a site whose earlier upgrade aborted before its permission pass, this was the root
cause - a re-deploy of 0.4.20+ (which normalises ownership) plus the doctor's --fix
verify now leaves audit.log writable.

## Reviewing what is audited (the full set)

- Operator mints a connect code -> user-onboarding-web (manager UI)
- Connector registers -> oauth-register
- Connect code redeemed / consent -> oauth-authorize (ok / fail)
- Access token issued -> connect
- Token renewed -> oauth-refresh
- Login / logout / claim / token exchange+rotate -> login, logout, claim-redeem,
  token-exchange, token-rotate (0.4.21)
- Content writes over UI / WebDAV / MCP -> create / edit / delete / move
- User / ACL / config / theme / layout changes -> the manager-api material events
- Reads / browsing -> NOT audited (access log / SM083 stats)
