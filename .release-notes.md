Release 0.5.12 - Group-based capabilities (Phase 1, backend)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

Groundwork for managing permissions by role instead of per account (SM095).

- Capabilities can now be carried by a GROUP; an account's effective permissions
  are the union across its groups. Resolved in the one place every surface already
  reads, so no call site changed.
- On first use a site seeds four role groups - Content manager, Appearance manager,
  AI site manager (includes analytics), User manager - and flags its existing
  manager group (lazysite-admins) as a full-capability manager group, so the
  operator keeps full access and assigns everyone else to groups from there.
- Non-breaking: existing per-user grants still apply this phase.

Next: Phase 2 adds the Groups page (per-group capability toggles, manager-group
switch, member add/remove); Phase 3 is the clean cut (capabilities from groups
only; per-user toggles removed from the Users page).
