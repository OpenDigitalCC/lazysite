Release 0.4.58 - Appearance page + per-layout layout management

Manage layouts the way themes were already managed. Full suite green (1751);
perlcritic clean; strict SBOM gate green.

## What's new

- The manager "Themes" tab is now "Appearance" (/manager/appearance; /manager/
  themes redirects). It is ordered: layouts-repo setting, active layout & theme
  switcher (moved off Config), upload a theme, browse the repo catalogue, and
  installed layouts & themes.
- Install a single layout and its theme(s) on demand from the repo's
  manifest.json (action_layouts_manifest / action_layout_install) - no more
  pulling the whole repo to use one layout.
- Delete a layout, which removes its themes too, behind a confirm and only when
  it is not the active layout; a recovery snapshot is kept (action_layout_delete).
- The active layout/theme switcher moved from Config to Appearance; Config shows
  them read-only with a link.
- Cross-layout theme preview now loads the correct CSS (the layouts repo migrated
  its layouts to the portable [% theme_assets %] link).

## Dependency

The per-layout install reads manifest.json from the layouts repo on branch
'layouts_ref' (default main). The matching packaging lands in lazysite-layouts
(manifest.json + releases/layouts/*.zip + portable assets); push/merge that to
the read ref, or set layouts_ref to its branch, for live installs.
