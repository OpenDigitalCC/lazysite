Release 0.4.46 - scan: custom front-matter passthrough (SM125)

From a CC field report building a dynamic theme gallery. Full suite green; perlcritic
clean; strict SBOM gate green.

## Self-describing scan: cards

`scan:` page objects exposed only a fixed field set (url/title/subtitle/date/tags), so a
registry card had to smuggle custom data through tags and the filename. Now:

- Any non-control front-matter key passes through onto the page object under its own
  name - [% t.kind %], [% t.demo %], [% t.accent %], [% t.order %]. Computed fields take
  precedence; control keys (layout/theme/auth/register/search/tt_*/_*) are excluded.
- Surrounding quotes are stripped (accent: "#7C5CFF" -> #7C5CFF; quote a value starting
  with # since a bare # ... is a YAML comment), and TT markers are stripped for safety.
- sort=<custom-key> works and is numeric-aware (sort=order gives 2 before 10).
- The recursive ** glob (scan:/gallery/**/*.md) was already supported and is now
  documented (the "one level only" note was stale).

A gallery becomes one recursive scan + sort=order + clean [% t.kind %] keys - no
tags-hack, no filename derivation.
