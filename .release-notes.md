Release 0.5.7 - AI visitor-log analysis, Phase 2 (audit)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

- The audit trail now requires the `analytics` capability - for API/token clients
  and for managers in the in-page Audit view alike. A manager without it sees
  "...requires the analytics permission. Ask the operator to grant it: lazysite-
  users.pl set <user> analytics on". Grant analytics to accounts that should see
  the audit trail (and the visitor stats).
- The Audit view reads through an append-only cache: only audit lines appended
  since the last load are parsed, not the whole log - the same incremental cache
  used by the visitor-stats export.
