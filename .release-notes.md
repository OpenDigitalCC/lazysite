Release 0.4.57 - nginx reload on deploy is now opt-in

Tidy-up after the editor fix. Shell-only change. Full suite green; perlcritic
clean; strict SBOM gate green.

## Change

The post-upgrade nginx reload added in 0.4.55 is now opt-in, off by default. It
was added for a stale-open_file_cache theory that was never confirmed - nginx's
open_file_cache is off by default and self-heals within its validity window when
on, and the actual blank-editor fault was an unrelated render bug (0.4.56). An
unconditional root-level nginx reload on every deploy isn't justified.

Enable it only if your nginx has open_file_cache enabled and you don't want to
wait out the validity window:

    LAZYSITE_RELOAD_NGINX=1 lazysite-hestia-update-all

The ?v=<version> cache-buster on the manager asset URLs is unchanged and still
covers the browser/CDN side on every deploy.
