Release 0.4.86 - Manager bar on the homepage

Full suite green; perlcritic clean; strict SBOM gate green.

The manager admin bar was missing on "/" (the homepage) on every site, while
showing on all other pages. Cause: the Apache template listed index.html in
DirectoryIndex, and that is exactly lazysite's homepage cache file - so Apache
served it directly for "/", bypassing the processor that injects the bar.

The lazysite-app web template now drops index.html from DirectoryIndex (a lazysite
home routes through the processor; a static index.html with no index.md is still
served directly).

IMPORTANT - this is an Apache template change. Existing sites need their vhost
rebuilt to pick it up:
  - refresh the template files:  lazysite-hestia-update-all.sh --tpl <STAGE>
  - rebuild a vhost to apply it:  v-rebuild-web-domain <USER> <DOMAIN>
Test ONE site first (confirm the homepage still loads AND now shows the bar for a
manager), then roll out to the rest.
