Release 0.5.24 - Faster Users page, one capabilities view, safe "move under"

Full suite green (1979 tests); perlcritic clean; strict SBOM gate green.  Channel: edge.

- The Users page now loads in a single request instead of three. Each request was
  a Perl cold start under plain CGI, and the old "whoami" call spawned two more
  subprocesses on top - so the page felt slow even with one account. One call now
  returns the accounts, their group-resolved capabilities, all groups, and your own
  identity together.
- Capabilities on a user's card are shown once: the channel x capability grid (the
  duplicate chip list has been removed). It is read-only - capabilities are edited
  on the Groups page - and re-checks when you open it, with a Recheck button.
- "Move under" (reparenting an account) no longer offers the account's own
  sub-tree, or itself, as a destination - those would make a cycle, which the
  server already refuses.
