## Multi-client AI partnering (SM076)

The OAuth + MCP server is now confirmed client-neutral - Claude.ai web, ChatGPT
(developer mode), Claude Code/Desktop all work on one implementation.

- Tool annotations: every tool declares readOnlyHint/destructiveHint/openWorldHint
  + an output schema. ChatGPT requires these (per-call approval + read/write
  gating); harmless elsewhere.
- The Users-page button is now "Connect an AI assistant" (was "Set up Claude.ai").
  The Step-1 card shows the URL + one-time connect code universally, with a
  per-app pointer (Claude.ai vs ChatGPT) and a link to the full guide. Once the
  connection authenticates, Step 2 reveals the no-secret task prompt.
- New /docs/ai-connector-setup (operator-gated): step-by-step for Claude.ai,
  ChatGPT (incl. its Plus/Pro read-only + slower caveats), the static-bearer path
  for Desktop/Code/scripts, a generic "any MCP client" section, and
  troubleshooting.
- The OAuth authorize endpoint logs client/redirect mismatches to aid debugging.

## Fix: processor paragraph-wrapped block HTML

Text::MultiMarkdown wrapped top-level block HTML (e.g. a hero <section>) into
invalid <p><section>...</section></p> - found by an AI partner reviewing a live
site. convert_md now unwraps the spurious <p>/</p> around block-level elements;
ordinary paragraphs are untouched. Affects every site.

## Feature: content vs theme capability (SM082)

New `manage_content` capability governs the content namespace, defaulting to the
existing `webdav` grant when unset (no change for current partners). Turn it off
(a new Users-page toggle) for a theme-only partner: content reads/writes are
refused while theme/layout work still functions. Enforced in both the MCP tools
and raw WebDAV.
