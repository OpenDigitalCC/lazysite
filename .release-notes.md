Release 0.5.17 - Route all capability checks through the resolver (SM095 c0)

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

Internal, non-breaking: the last places that read capabilities from per-account
settings - the sub-user creation gate and the connector onboarding briefs - now
go through the single resolver, so a capability granted via a GROUP applies to
them too (e.g. a user-managers member can create sub-users; an agent group's
partner brief lists its real capabilities). Adding a user to a group also seeds
the default role groups so their capabilities take effect. Behaviour is otherwise
unchanged; this is the groundwork for the groups-only clean cut.
