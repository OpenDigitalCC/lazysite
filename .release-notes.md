Release 0.5.11 - Grant analytics from the UI + documentation

Full suite green; perlcritic clean; strict SBOM gate green.  Channel: edge.

- The "Analytics (visitor stats + audit)" permission is now a toggle on the Users
  page (previously only settable from the command line). Tick it on the account
  that should see the visitor stats / audit trail, and save.
- whoami advertises the analytics capability, so a granted AI connector knows it
  may call analyse_visitors.
- The audit "permission required" message now points to the Users page, not a CLI
  command.
- AI/MCP docs updated: the analyse_visitors tool and the analytics capability are
  in the connector-tools reference and the AI-partner manifest, onboarding lists
  "analyse visitor trends", and the manager guide notes the audit gating. The
  /docs/ai-briefing-stats briefing (shipped 0.5.6) tells the AI how to use it.
