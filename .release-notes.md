Release 0.4.34 - Files sortable+paginated; generator meta; config toggles; dark polish

Feature work (SM111/SM112) plus the next slice of UI polish (SM114) and dark-mode
follow-ups from the live review. Full suite green; perlcritic clean; strict SBOM gate
green.

## File manager: sortable + paginated (SM111)

Name / Access / Modified column headers sort on click (toggling direction, with an
indicator); directories with many files paginate at 50/page with a pager. The list is
now rendered data-driven, so the type/name filter, the sort and the paging all compose
(previously the filter hid DOM rows, which would not have paged).

## Generator meta (SM112)

Generated pages now carry <meta name="generator" content="lazysite X.Y.Z"> (version
from the install state), plus author/description from front matter when present. It is
injected into the head after the layout renders, so it works with any installed
layout. Opt out per site with meta_generator: false in lazysite.conf.

## Config toggles + disable warning (SM114, part)

Boolean site settings (Manager, WebDAV publishing, searchable-by-default) are now
switches instead of enabled/disabled dropdowns. Saving with the manager set to
disabled confirms first ("locks the manager UI for everyone, including you"). The
editor breadcrumb root is a files icon rather than "/".

## Dark-mode polish (live feedback)

Text is brighter in dark mode for stronger contrast, and a baseline ensures every form
control - including previously unstyled ones like the theme browse - inverts with the
theme (tokenised background/text/border/focus).

## Captured

SM115 (form submissions: a view link, edit-safety/concurrency so a submission can't be
lost to an open editor, and a form-submission audit event with a blank user), SM116
(a readable dark colour scheme for the CodeMirror markdown editor). Remaining SM114:
the manager-groups picker, logical grouping of site settings, and a unified breadcrumb
on the Files page.
