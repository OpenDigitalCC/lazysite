## Connector tools

- preview_page(path) - server-side render (fresh, LAZYSITE_NOCACHE, no cache
  write) returning HTML + status, so verification stays in-channel. Public-view;
  a protected page shows the auth gate; capped like read_file.
- whoami echoes the full tools[] manifest - the agent discovers every tool in
  one call (the connector loads tools a few at a time).
- copy_file(from, to) - template a new page from an existing one.
- get_permissions(path) - read a path's ACL before set_permissions.
- list_form_handlers + bind_form(form, handler) (SM088) - wire a form to delivery
  by referencing an operator-vetted handler; the connector never sets a
  destination or credential (those stay operator-only).

## Clearer errors + cache correctness

- 401 distinguishes sign-in-incomplete vs credential-invalid/expired
  (error.data.reason); error responses carry a machine-readable kind.
- A nav.conf save clears ALL page caches (nav is on every page) and flags
  cache_rebuilt - fixes "edited nav, verified one page, wrongly assumed
  site-wide".

not Claude.ai / ChatGPT web (OAuth-only; use Connect an AI assistant), and that
